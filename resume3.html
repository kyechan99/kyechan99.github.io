INTRODUCE

소프트웨어 개발은 문제를 개선하고 변화시키는 일에 큰 가치가 있다고 느낍니다.

이전에 겪었던 실수와 고민을 반복하지 않기 위해 문제 해결을 위한 프로젝트를 진행하거나 공유합니다.

소통과 협업은 문제를 공유하는 상황에서 힘이 나온다고 생각하며, 귀 기울이며 수용하고 배우고자 노력합니다.

일일 사용량 45,000 건의 오픈소스를 관리하고 있습니다.

기술의 사용법에 능숙해지기보다, 왜 필요하고 무엇을 해결하는지 근원적 물음을 제기합니다.

이를 바탕으로 오픈소스에 관심이 많아 분석해보거나 개발 해보며 다양하게 도전해오고 있습니다.

특히 개발자에게 필요한 기능이나 문제를 경험하면서, 직접 개선하고 변화시키는 일이 즐겁습니다.

Scribubble

WebVR을 활용한 3D 공간 Drawing

Preact Three.js A-Frame Javascript styled-component

Links

  • GitHub: https://github.com/scribubble/scribubble
  • Service: https://github.com/scribubble/scribubble

Service

  • 팀 구성원 모두 Three.js 를 처음 접하게 되면서 매주 학습 내용을 공유/토론함
  • 스터디를 통해 도형 추가나 작업물 선택등의 기능이 고안되며 서비스에 추가됨
  • 초기 프로젝트 설계와 확장성을 고려한 개발로 웹 뿐만 아니라 VR 환경도 지원할 수 있게됨

Challenge

  • 프로젝트 경량화
    • 서비스 요구사항 대비 불필요한 React 도입 고려
    • 대체안 Preact 도입으로 인한 효과를 정리하고 팀원들과 토론하여 설득함
    • React 에서 Preact 로 작업 결정하면서 렌더링 속도와 메모리 사용량 개선 시도
    • VR 환경에서의 사용도 고려 가능해지면서 WebVR 기능 추가가 가능해짐
  • 프레임 저하 개선
    • 평균 초당 프레임(FPS) 60에서 Drawing 작업시 10 FPS 이하로 저하됨
    • 데이터가 입력에 맞추어 동적 증가되도록 하고 재렌더링 요청 작업을 불필요하게 만들면서 60 FPS 정상화
  • 브라우저 환경간 선 굵기가 호환 문제 해결
    • Three.js 의 공식 Line 객체가 모든 브라우저에 호환되지 못하는 문제 발생
    • 라이브러리의 샘플 단계 Line2 와의 비교 분석을 통해 프로젝트에 도입하면서 브라우저 호환함
  • 3D 상의 Drawing 좌표계 문제 해결
    • 무한한 공간에서의 깊이와 시점으로 인한 좌표 변경을 카메라와 평행한 평면과 event와의 꼭짓점 계산으로 고정된 좌표를 얻어냄
